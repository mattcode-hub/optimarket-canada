USER TYPES:
===========

interface User {
  id: string;
  name: string;
  email: string;
  avatar: string;           // Generated initials
  phone: string;
  location: string;
  bio: string;
  memberSince: string;      // ISO date string
  isSeller: boolean;        // Determines dashboard access
}

ROUTES:
=======

Public Routes:
- /auth/login          Login page
- /auth/signup         Registration page
- /                    Homepage (header shows auth state)

Protected Routes:
- /auth/profile        User profile (redirects to login if not auth)
- /dashboard           Seller dashboard (visible if isSeller = true)

HOOKS:
======

useAuth() - Access authentication state and methods
  Returns: {
    user: User | null
    isAuthenticated: boolean
    isLoading: boolean
    login(email, password): Promise<void>
    signup(data): Promise<void>
    logout(): void
    updateProfile(data): Promise<void>
  }

VALIDATION:
===========

Login:
- Email required
- Password required

Signup:
- Name required (non-empty)
- Email required and must be valid format
- Password required (min 6 characters)
- Confirm password must match
- Location required
- Phone optional
- Terms must be accepted

Profile Update:
- Name can be edited
- Phone can be edited
- Location can be edited
- Bio can be added/edited

IMPLEMENTATION NOTES:
=====================

1. localStorage is wrapped in useEffect to avoid SSR hydration issues
2. Demo accounts are pre-seeded in DEMO_ACCOUNTS array
3. New signups add to DEMO_ACCOUNTS array (in-memory storage)
4. User avatars are generated from initials (first letter of each name word)
5. memberSince dates are auto-generated as ISO date strings
6. Password is NOT stored in localStorage (only in DEMO_ACCOUNTS)
7. Auth state persists across page refreshes via localStorage
8. All forms have loading states and error handling
9. Dropdown menus close when clicking links
10. Mobile menu updates to show auth options dynamically
